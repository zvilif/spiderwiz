API Overview

A Spiderwiz-based application defines its network connections through its configuration file.The current version of the framework supports three types of communication – TCP/IP, WebSockets and disk files (mainly for debugging purposes). The first two can be defined as either a client or a server. An application that is configured as a WebSocket server needs to include the spiderwiz-websocket-server module as a dependency. The module hooks up an Endpoint and a Filter that are necessary for the server to work.

The classes that are included in this module are loaded automatically when the application starts. There is no need to access them directly, therefore they are not documented in this Javadoc.
